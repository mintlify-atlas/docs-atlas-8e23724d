The fastest way to experience DataOps TestGen is through the built-in demo. The demo installs TestGen as a Docker Compose application, loads a sample dataset, runs data profiling, generates test suites, executes the tests, and populates the anomaly monitoring dashboards — giving you a complete picture of the platform without connecting to a real database.

<Note>
  The demo requires Docker and Python 3.9 or later. If you want to skip the demo and go straight to connecting your own database, see [Install with Docker](/installation/docker).
</Note>

## Prerequisites

Before you begin, verify that Docker and Docker Compose are installed and running on your machine.

```bash
python3 --version       # 3.9, 3.10, 3.11, or 3.12
docker -v               # 26.x or later
docker compose version  # 2.32 or later
```

Docker Desktop on Mac and Windows includes Docker Compose. On Linux, install the Docker Compose plugin separately if needed.

## Run the demo

<Steps>
  <Step title="Create a working directory">
    Create a new, empty directory to hold the installer and generated files, then navigate into it.

    ```bash
    mkdir testgen-demo && cd testgen-demo
    ```
  </Step>

  <Step title="Download dk-installer">
    Download the installer script using `curl`. All subsequent commands must be run from this directory.

    ```bash
    curl -o dk-installer.py 'https://raw.githubusercontent.com/DataKitchen/data-observability-installer/main/dk-installer.py'
    ```

    <Tip>
      If `curl` is not available, download `dk-installer.py` manually from the [data-observability-installer](https://github.com/DataKitchen/data-observability-installer) repository and place it in this directory.
    </Tip>
  </Step>

  <Step title="Install TestGen">
    Run the installer. It downloads the Docker images and starts a Docker Compose application. This step typically takes 5–10 minutes depending on your network connection.

    ```bash
    python3 dk-installer.py tg install
    ```

    When the command completes, the output shows the URL and credentials for the TestGen UI.
  </Step>

  <Step title="Load the demo dataset">
    Run the demo setup command. This loads a sample database, runs data profiling, auto-generates a test suite, executes the tests, and enables anomaly monitors.

    ```bash
    python3 dk-installer.py tg run-demo
    ```
  </Step>

  <Step title="Open the TestGen UI">
    Open the URL shown in the installer output in your browser — by default, [http://localhost:8501](http://localhost:8501). Log in with the credentials displayed at the end of the `tg install` step.

    You will see data profiling results, a generated test suite with executed results, and populated anomaly monitoring dashboards for the sample dataset.
  </Step>
</Steps>

## What the demo shows

The demo data populates the full TestGen workflow:

- **Profiling results** — column-level statistics including value distributions, nullability rates, uniqueness, and data patterns across the sample tables.
- **Auto-generated tests** — hundreds of data quality validation tests covering completeness, consistency, validity, uniqueness, and statistical thresholds, generated automatically from the profile.
- **Test results** — executed test outcomes with pass/fail status and drill-down details for any failures.
- **Anomaly monitoring** — freshness, volume, and schema drift monitors tracking changes over simulated data refreshes.

## Clean up demo data

After exploring the demo, you can remove the demo data without uninstalling TestGen.

```bash
python3 dk-installer.py tg delete-demo
```
