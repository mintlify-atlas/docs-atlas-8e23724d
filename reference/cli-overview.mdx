The `testgen` CLI gives you full programmatic control over DataOps TestGen — from initializing the system database to running profiling jobs, executing test suites, and exporting results. You can use it directly on a host machine after installing the Python package, or run it inside the official Docker container.

## Installation

<Tabs>
  <Tab title="pip">
    Install the `dataops-testgen` package from PyPI. Python 3.12 or later is required.

    ```bash
    pip install dataops-testgen
    ```

    After installation the `testgen` command is available on your `PATH`.
  </Tab>
  <Tab title="Docker">
    The official image ships with `testgen` pre-installed. Pass any command as the Docker `CMD`:

    ```bash
    docker run --rm datakitchen/dataops-testgen:latest testgen --help
    ```
  </Tab>
</Tabs>

## Global flag

The only flag that applies to every command is `--verbose`. It must appear **before** the subcommand name.

| Flag | Short | Default | Description |
|------|-------|---------|-------------|
| `--verbose` | `-v` | `false` | Enables debug-level logging for the entire run. |

```bash
testgen --verbose run-profile --table-group-id <id>
```

<Warning>
  Placing `-v` after the subcommand name has no effect. Always put it immediately after `testgen`.
</Warning>

## Commands by category

### Application

Commands for running and initializing the TestGen application itself.

| Command | Description |
|---------|-------------|
| [`run-app`](/reference/cli-data#run-app) | Runs TestGen application modules (`all`, `ui`, `scheduler`, `mcp`). |
| [`setup-system-db`](/reference/cli-data#setup-system-db) | Initializes the TestGen PostgreSQL system database. |
| [`upgrade-system-version`](/reference/cli-data#upgrade-system-version) | Upgrades the database schema to the current version. |
| [`quick-start`](/reference/cli-data#quick-start) | Loads sample data for a demo environment. |
| [`mcp-token`](/reference/cli-data#mcp-token) | Generates a JWT for MCP server authentication. |

### Profiling

Commands for profiling data sources and inspecting profiling results.

| Command | Description |
|---------|-------------|
| [`run-profile`](/reference/cli-profiling#run-profile) | Profiles a table group and stores the results. |
| [`list-table-groups`](/reference/cli-profiling#list-table-groups) | Lists all table groups in a project. |
| [`list-profiles`](/reference/cli-profiling#list-profiles) | Lists all profile runs for a table group. |
| [`get-profile`](/reference/cli-profiling#get-profile) | Fetches column-level details for a profile run. |
| [`get-profile-anomalies`](/reference/cli-profiling#get-profile-anomalies) | Fetches anomalies detected during a profile run. |

### Test management

Commands for generating, running, and monitoring tests.

| Command | Description |
|---------|-------------|
| [`run-test-generation`](/reference/cli-testing#run-test-generation) | Generates or refreshes tests for a test suite. |
| [`run-monitor-generation`](/reference/cli-testing#run-monitor-generation) | Generates or refreshes monitors for a monitor suite. |
| [`run-tests`](/reference/cli-testing#run-tests) | Executes tests defined in a test suite. |
| [`run-monitors`](/reference/cli-testing#run-monitors) | Executes monitors defined in a monitor suite. |

| [`list-profiles`](/reference/cli-profiling#list-profiles) | Lists profile runs for a table group. |

## Output behavior

Most listing and detail commands write their results to a CSV file in the current directory (for example, `list_profiles.csv`). Pass the `--display` / `-d` flag to also print the table to your terminal.

```bash
testgen list-test-suites --display
```

## Automatic schema upgrades

If your system database schema is outdated, TestGen automatically runs `upgrade-system-version` before executing the requested command. You can suppress this behavior by running `upgrade-system-version` manually ahead of time.
